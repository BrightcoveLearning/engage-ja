    カスタムドメインおよびSSLを設定する

    このトピックでは、Web エクスペリエンスのカスタムドメインと SSL を構成する方法について説明します。

    Web エクスペリエンスを初めて作成すると、の形式でランダムドメインが割り当てられます<site-slug>.bcvp0rtal.com 。たとえば http://site-148393.bcvp0rtal.com となります。エクスペリエンスが公開されると、閲覧者はこのドメインのサイトにアクセスできるようになります。Brightcove Engage は、ブライトコーブが提供するドメインまたは独自のカスタムドメインで Web エクスペリエンスを公開する機能を提供します。

    Web エクスペリエンスを複数の URL に割り当てることもできます。これにより、パブリッシャーはサイトを複製することなく、同じサイトを異なる機能に使用できます。たとえば、キャンペーンやメインの動画ポータルとして使用したい動画サイトがある場合、異なる URL をブランド化して、それらをすべて同じサイトに割り当てることができます。サイトの複製は必要ありません。また、URLは必要でなくなれば簡単に削除できます。

    ブライトコーブ提供ドメインを使用する

    ブライトコーブが提供するドメインを使用すると、IT 部門と協力してカスタム CNAME を作成したり、レジストラに新しいドメイン名を登録したりすることなく、ウェブエクスペリエンスのドメインをすばやく簡単に作成できます。これらは「箱から出て」利用可能で、ボタンをクリックするだけで実装できます。使用可能なドメインは次のとおりです。

    • brightcoveengage.dove
    • bcvp0rtal.com (デフォルトでウェブエクスペリエンスに割り当てられている)

    これらはすべてSSLをサポートします。SSLサイトにアクセスするには、URLにhttpsを追加するだけです。例： https://mysite.brightcoveengage.video

    （パブリッシャーが登録した）カスタムドメインを使用する

    Web エクスペリエンスは、登録した独自のカスタムドメインでホストすることもできます。URLをブランディングし、現在のWebサイト/URLの命名構造と一致させることができます。これには、ドメイン名を登録し、その後、ホスティングプロバイダーとカスタムCNAMEレコードを設定する必要があります。CNAMEレコードを設定する方法については、下のセクションを参照してください。

    カスタムドメインを作成する

    カスタムドメインを作成するには、次の手順を行います。

    1. ブライトコーブのエンゲージメントを開きます。
    2. 「プロジェクト」ページを開き、ページの上部にある「設定」リンクをクリックします。
    3. 左側のナビゲーションで、[ カスタムドメインと SSL ] リンクをクリックします。
    4. [カスタムドメインの作成]をクリックします。
    5. 使用するドメインオプションを選択します。
      • brightcoveengage.Video -サイトドメインの形式になります<your domain>.brightcoveengage.video。
      • [ カスタム ]: 自分のカスタムドメインでサイトをホストするときに選択します。このオプションを選択すると、ホスティングプロバイダーとCNAMEレコードを設定することも必要になります。
    6. ドメイン名を入力します。ドメインが有効か無効かを示すメッセージが表示されます。ドメイン名が無効の場合は、新しいドメイン名を入力します。
    7. ブライトコーブが提供するドメインを使用してサイトを公開する場合は、このドメインに SSL を適用するためのオプションがあります。閲覧者がサイトにアクセスした場合に、非SSLバージョンのサイトからのリダイレクトを有効にするには、［このドメインに SSL を適用する］をオンにします。
    8. 自身のカスタムドメインにサイトを公開する場合、SSLを使用してサイトのセキュリティーを確保するオプションがあります。SSLを使用してサイトのセキュリティーを確保する場合は、[SSLの使用]チェックボックスを選択します。SSL の使用については、「SSL を使用した Web エクスペリエンスの保護」を参照してください。
    9. この URL を別のURLにリダイレクトするには、[リダイレクトを有効にする ] をオンにします。通常、これはこのドメインがしばらく使用された後にのみチェックされます。このオプションを選択すると、この既存の URL をリダイレクトするリダイレクト先の URL を入力します。 [ 永続的なリダイレクトを使用する (HTTP ステータスコード 301) ] をオンにして、リダイレクトが永続的であることを示します。検索エンジンのスパイダーは、Webページの応答ヘッダーに301ステータスコードを見つけると、それはこのWebページが存在しなくなったことを理解し、新しいものとインデックス付きURLを置き換えます。このオプションでは、古い Web ページのトラフィックとページランクが保持されます。
    10. [保存]をクリックします。設定されたドメインのリストが表示されます。
      注：ブライトコーブ提供ドメインを使用する場合、SSLステータスは常に提供になります。これは、サイト URL の先頭に https://を追加すると、ブラウザと Brightcove Engage サイト間のデータ転送が安全であることを示しています。
    11. 必要に応じて他のドメインを作成します。

    ドメインを編集または削除するには、編集（）ボタンまたは削除（）ボタンをクリックします。

    カスタムドメインを作成したら、エクスペリエンスのプロパティの一部としてエクスペリエンスに割り当てることができます。詳細については、「カスタムドメインを Web エクスペリエンスに割り当てる」を参照してください。

    お客様のホスティングプロバイダーでCNAMEレコードを構成する

    注：CNAMEレコードの設定が必要なのは、カスタムドメインをサイトに割り当てるときだけです。ブライトコーブ提供ドメインを使用するサイトには、CNAMEレコードを作成する必要は「ありません」。

    CNAME（正規名）レコードを設定する手順は、お客様のホスティングプロバイダーによって異なります。使用する各カスタムドメインに、CNAMEレコードを設定する必要があります。サイトが SSL を使用するように設定されている場合、CNAME レコードは CloudFront URL をポイントします。詳細は、「SSL を使用した Web エクスペリエンスの保護」を参照してください。CNAME として構成するには、カスタムドメインにサブドメインを含める必要があることに注意してください。たとえば、CNAME brightcovelearning.com を設定できるように、engage 前に配置する必要があります。

    CNAME 値は、カスタムドメインと SSL 設定の一部として表示されます。

    この例では、engage.brightcovelearning.com のカスタムドメインの場合、CNAME レコードを追加し、engage のホストを追加して、それを 5e84f4c7d58950766328a37d.bcvp0rtal.com にポイントします。

    この手順については、ネットワーク管理者または Web ホスティングプロバイダーに問い合わせる必要があります。DNSの変更は最大で48時間かかることがあります。

    SSLを使用してサイトのセキュリティーを確保する

    注：SSL 証明書を使用する場合は、いかなる場合でもプライベートキーを共有したり、ブライトコーブサポートに送信したりしないでください。

    SSL  を使用したポータル・エクスペリエンスのセキュリティ保護には、次の 2 つのオプションがあります。

    1. 割り当てられたサイトスラッグを使用して公開されているサイト（例<site-slug>.bcvp0rtal.com）、または Brightcove 提供ドメイン（<site>.brightcoveengage.video）を使用しているサイトでは、https をサポートするための設定は必要ありません。URL の先頭に https://を追加すると、ブラウザとエクスペリエンスの間のデータ転送がセキュリティで保護されます。また、[ このドメインに SSL を強制 (推奨) ] を選択して、ビューアがサイトにアクセスした場合に、非 SSL バージョンのサイトからのリダイレクトを有効にすることもできます。
    2. カスタムドメインでホストされているサイトの場合は、ドメイン名を入力するときに [ SSL を使用 ] オプションをオンにします。証明書情報を入力するよう表示があります。詳細については、「 SSL を使用した Web エクスペリエンスの保護」を参照してください。
