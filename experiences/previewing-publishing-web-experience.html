Web エクスペリエンスのプレビューと公開

このトピックでは、Web エクスペリエンスをプレビューして公開する方法について説明します。

Web エクスペリエンスは、開発中に簡単にプレビューできます。体験をプレビューするためのリンクも共有できます。エクスペリエンスが完了したら、公開して一般公開することができます。

Web エクスペリエンスのプレビュー

Web エクスペリエンスをプレビューすると、モバイル、タブレット、デスクトップデバイスでエクスペリエンスがどのように表示されるかを確認できます。Web エクスペリエンスをプレビューするには、エクスペリエンスを編集し、エクスペリエンスエディターで [ プレビュー ] をクリックします。

次の 2 つのオプションがあります。

  • レスポンシブプレビュー -プレビュー用に新しいブラウザウィンドウにエクスペリエンスを開きます
  • 共有可能なリンクを取得 -プレビューを他のユーザーと共有できるように、プレビュー URL を提供します。

生成された共有可能な URL は 72 時間有効です。

注:アクセスコントロール設定の一部としてセッションタイムアウトを構成した場合、72 時間のタイムアウトは設定されたセッションタイムアウトよりも優先されます。エクスペリエンスが公開されると、セッションタイムアウト（存在する場合）が適用されます。

Web エクスペリエンスの公開

Web エクスペリエンスを公開すると、閲覧者が利用できるようになり、サイトの静的 URL が生成されます。エクスペリエンスを公開するには、次の手順に従います。

注:最近動画のメタデータやプレイリストに変更を加えた場合、公開済みのエクスペリエンスにそれらの変更が表示されるまでに数分かかる場合があります。
  1. エクスペリエンス名をクリックして、エクスペリエンスエディターでサイトを開きます。
  2. [ パブリッシュ] をクリックします。
  3. パブリッシュの確認ダイアログで、[ 今すぐ公開] をクリックします。
  4. エクスペリエンスが公開され、サイトの URL が表示されます。

NS bcvp0rtalドメインはデフォルトでSSLをサポートしているため、以下を使用してサイトにアクセスすることもできます。 https：//、セットアップやSSL証明書は必要ありません。

エクスペリエンスが公開されると、URL はプロジェクトホームページのサイト詳細の一部として表示されます。

Web エクスペリエンスへの変更の公開

エクスペリエンスに変更を加える場合は、そのエクスペリエンスを再公開する必要があります。再公開する必要があるエクスペリエンスには、未公開の変更のエクスペリエンスのステータスが表示されます。

変更を再公開するには、このトピックの「 Web エクスペリエンスの公開」セクションで説明されている手順を実行します。

注:変更が表示されない場合は、ブラウザのキャッシュと Cookie をクリアする必要がある場合があります。

ウェブ体験をオフラインにする

エクスペリエンスをオフラインにすると、その URL では使用できなくなります。エクスペリエンスをオフラインにするには、行をクリックし、[ オフラインにする] をクリックします。エクスペリエンスをオフラインにする必要があることを確認するには、[ オフラインにする] をクリックします。エクスペリエンスを再び利用できるようにするには、エクスペリエンスを再公開する必要があります。エクスペリエンスがオフラインになると、その URL 経由でアクセスできなくなります。

注:エクスペリエンスをオフラインにしても、エクスペリエンスが削除されません。
